# SPEC.md

Rust製の静的サイトジェネレータ

- 目的:  
  - `Pages/`に`Pages/<カテゴリ>/.../*.md`を放り込んでpushするだけでページの追加とサイトの更新をできるようにする  
  - 記事の作成日時と更新日時を表示できるようにする
  - トップページなどの特別なものは自動生成せず、HTMLをハードコードする
- 入力: ユーザーが唯一編集するMarkdown群（`Pages/`）
- 出力: 生成されたHTML等（`Meta/Site/`）
- カテゴリ: `Pages/`直下のフォルダ名（例: `研究ログ`）
- サブカテゴリ: カテゴリ配下の任意階層（例: `研究ログ/2026`）
- slug: URL用の安全文字列（表示は日本語、URLは安全文字列）

## ディレクトリ構造

```directory
Pages/                 # 入力（Markdown原本）
  研究ログ/
    AIの指示に100%従うチャレンジしたら全裸でベランダでコーディングする羽目になった話.md
Special/               # 特別ページ（HTML直書き）
  home.html            # トップページ
  404.html             # 404ページ
static/                # 静的資産
  base.css
  article.css
  icons/
    calendar.svg
Meta/                  # 出力（生成物）
  site/                # GitHub Pagesにデプロイする成果物
  cache/               # 差分ビルド用
```

## 入力ファイル仕様

- 拡張子は`.md`のみ  
- カテゴリ：`Pages/` 直下の第一階層フォルダ名  
- サブカテゴリ：カテゴリ配下の相対パス（階層は任意）  

## Frontmatter仕様（Yaml）

- Yaml frontmatterをサポート
- 区切りは`---`（先頭にある場合のみ認識）

```yaml
---
title: AIの指示に100%従うチャレンジしたら全裸でベランダでコーディングする羽目になった話
tags: [快感, ほぼ犯罪]
draft: false
slug: Exhibitionist-coding
summary: 1行目の要約（任意）
---
```

- パース失敗時はページを表示しない

## slug仕様

### slugifyルール

- Unicode正規化
- 小文字化
- 空白→`-`
- 許可文字: `[a-z0-9-]`
- 先頭と末尾の`-`は削除
- 結果が空文字の場合は`p-<hash8>`にフォールバック(ハッシュはタイトルから生成)

## URL形式

- ディレクトリ型URL
- 実体は`index.html`

## 日付仕様

### 作成日時

定義: ファイルがGitに初めて追加された日時

### 更新日時

定義: 該当ファイルの最新コミット日時

### 表示規則

デフォルトで更新日時は表示されない。作成日時と更新日時が異なる場合に限り初めて表示される。  

## ビルド手順

1. 設定のロード
2. 出力ディレクトリ準備（必要ならクリーン処理）
3. `Static/`のコピー
4. `Pages/`を走査し対象の`.md`を収集
5. frontmatter + 本文読み込み
6. 作成/更新日時解決
7. slug決定（衝突チェック）
8. Markdown→HTML変換
9. 個別ページ作成
10. manifest出力
